# 北大法宝 MCP：Case Memo（类案检索报告/办案备忘）

这个 Skill 是**案例检索后的交付层**。
它的目标不是把案例结果堆成清单，而是把样本整理成一份可讨论、可流转、可追溯的类案备忘。

## 先看边界

- 只适用于需要形成“样本 + 比较 + 结论线索”的类案备忘场景。
- 如果用户只是想查单个案号、找某条案例原文，优先用更窄的检索或案号工具，不要强行上升成备忘。
- 不得把少量样本写成普遍裁判规则，不得无依据推断“法院倾向”。

## 这个 Skill 真正要完成的动作

把争议问题压缩成：

- 检索范围
- 核心类案样本
- 对我方有利/不利线索
- 样本局限
- 可追溯链接版本

## 缺信息先追问

以下信息缺失时，先补充，不要急着下“裁判倾向”：

- 争议焦点或核心法律问题
- 想支持哪一方立场
- 是否限定法院层级、地区、时间范围
- 备忘用途：内部研判、客户沟通、庭前准备

如用户只给了笼统问题，先把检索目标缩成一句话，例如“确认某类解除理由在近年裁判中的支持情况”。

## 推荐工作流

1. 确定检索问题：争点、法院层级、时间范围。
2. 选择路径：明确关键词时优先 `case-keyword`；案情描述较长时优先 `case-semantic`。
3. 筛样本：取最相关案例并记录命中理由，不要只堆数量。
4. 案号核验：对关键案例案号做二次识别核查。
5. 结构化总结：提炼有利点、不利点、适用边界、样本局限。
6. 链接增强：需要转发时，用 `doc-link` 输出可追溯版本。

## 输出纪律

- 不把有限样本概括成“法院普遍如此”。
- 不把“有利样本较多”写成“结果一定如此”。
- 对样本范围、地区偏差、时间跨度要单独提示。
- 若要支持某一方案，仍要同时列出主要不利点。

## 质量门槛

- “核心类案”必须保留可追溯线索（案号、链接或来源字段）。
- 不输出“普遍支持”“一贯观点”等绝对判断，除非样本充分且有依据。
- 对争议较大的结论，标注“需扩大样本继续检索”。

## 失败与降级

- 样本过少：明确写“仅找到有限样本，不能据此概括整体倾向”。
- 案号不稳：标记“待复核”，不要把该案例作为核心样本。
- 需要快速内部讨论：先给“Top 3 样本 + 有利/不利点”，再补完整备忘。
