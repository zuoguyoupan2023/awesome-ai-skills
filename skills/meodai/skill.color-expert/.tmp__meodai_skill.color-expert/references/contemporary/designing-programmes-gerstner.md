## Core Idea

Gerstner's argument is simple and still useful: designers should not aim only for single fixed solutions, but for **programmes** - structured systems that can produce many valid solutions.

Instead of solving one problem once, define the logic that can generate a family of solutions.

## Why It Belongs Here

This book is not a color-science text, but it is highly relevant to computational and systematic color work:

- palette systems
- generative art workflows
- design tokens
- rule-based variation
- algorithmic composition

It provides the design-method background for turning color decisions into reproducible, tunable procedures.

## Key Ideas

- **Programme over prescription:** specify relations and rules, not just one artifact
- **Morphology and logic:** design can be explored systematically through parameterized variation
- **Grid and method:** visual outcomes become more robust when structure is explicit
- **Variation without chaos:** a strong system makes experimentation cheaper and more coherent

## Why It Matters for the Skill

- Supports the repo's preference for **actual palette-generation systems** over random swatch picking
- Connects directly to tools like RampenSau, Poline, pro-color-harmonies, and generative palette formulas
- Useful framing when building color tooling that should output a space of solutions rather than one frozen answer

## Practical Takeaways

- When designing a color tool, describe the **parameters and constraints**, not only the final palette.
- Good palette generators are often better understood as **programmes for solutions**.
- If the output is hard to vary without breaking, the underlying program is probably too weak.
