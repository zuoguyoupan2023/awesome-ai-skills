## 一、核心概念

### 1.1 证据论证链的定义

证据论证链是指将法律主张分解为构成要件，再将具体证据逐一对应到各构成要件上，形成"**主张 → 要件 → 证据 → 证明力**"的完整逻辑链条。

```
┌─────────────────────────────────────────────────────┐
│                    最终法律主张                        │
│              （如：被告应赔偿损失）                     │
├─────────┬─────────┬─────────┬───────────┤
│  要件A   │  要件B   │  要件C   │  要件D     │
│ 违约事实  │ 损害结果  │ 因果关系  │ 归责事由   │
├─────────┼─────────┼─────────┼───────────┤
│ 证据A1   │ 证据B1   │ 证据C1   │  证据D1    │
│ 证据A2   │ 证据B2   │ 证据C2   │  (待补充)  │
│ 证据A3   │          │          │            │
├─────────┼─────────┼─────────┼───────────┤
│ 证明力:强 │ 证明力:强 │ 证明力:中 │ 证明力:弱  │
└─────────┴─────────┴─────────┴───────────┘
```

### 1.2 关键术语

| 术语 | 定义 | 说明 |
|------|------|------|
| **论证主张** | 当事人希望法院认定的法律结论 | 如"被告构成违约""原告有权解除合同" |
| **构成要件** | 法律主张成立所必须满足的各项法律条件 | 由实体法规范确定 |
| **证明对象** | 需要用证据加以证明的具体事实 | 每个构成要件可能对应一个或多个证明对象 |
| **证据材料** | 用于证明案件事实的各种材料 | 书证、物证、证人证言、电子数据等 |
| **证明目的** | 某项证据拟证明的具体事实命题 | 一项证据可有多个证明目的 |
| **证明力** | 证据对待证事实的证明程度 | 强/中/弱三级或更细化的评估 |
| **证据链** | 多项证据相互印证形成的证明体系 | 单一证据不足时需要证据链 |
| **证据缺口** | 某构成要件缺乏充分证据支撑的状态 | 需要补充证据或调整论证策略 |
| **举证责任** | 对特定事实承担提供证据并加以证明的责任 | 决定证据不足时的不利后果归属 |

### 1.3 证据论证链的层级结构

```
第一层：总主张（Ultimate Claim）
  │
  ├── 第二层：子主张/争议焦点（Sub-claims）
  │     │
  │     ├── 第三层：法律构成要件（Legal Elements）
  │     │     │
  │     │     ├── 第四层：待证事实（Facts to Prove）
  │     │     │     │
  │     │     │     ├── 第五层：证据材料（Evidence）
  │     │     │     │     │
  │     │     │     │     └── 第六层：证明力评估（Probative Value）
```

## 二、完整工作流程

### 阶段一：主张梳理与要件分解

**目标：** 明确全部法律主张，并将每个主张分解为法律构成要件。

#### 步骤1：识别与列举全部法律主张

- 从当事人的诉讼请求或抗辩理由出发
- 区分主要主张与辅助主张（备位主张）
- 明确每个主张的法律基础（请求权基础/抗辩权基础）

```
【操作模板】
主张编号：C-01
主张内容：被告应向原告支付违约金50万元
法律基础：《民法典》第577条、第585条
主张类型：□主要主张 / □备位主张
举证责任方：原告
```

#### 步骤2：分解法律构成要件

- 根据法律规范（请求权基础规范）提取构成要件
- 区分积极要件（需要主张方证明）与消极要件（需要对方反证）
- 标注举证责任分配

```
【操作模板】
主张C-01的构成要件分解：
┌──────┬──────────────┬──────────┬──────────┐
│要件编号│    要件内容     │ 举证责任方 │  要件性质  │
├──────┼──────────────┼──────────┼──────────┤
│ E-01  │ 合同有效成立    │   原告    │  积极要件  │
│ E-02  │ 被告存在违约行为 │   原告    │  积极要件  │
│ E-03  │ 违约金条款有效   │   原告    │  积极要件  │
│ E-04  │ 违约金数额合理   │   被告反证 │  消极要件  │
└──────┴──────────────┴──────────┴──────────┘
```

#### 步骤3：确定待证事实

- 将每个构成要件转化为具体的待证事实命题
- 区分无争议事实与有争议事实
- 对有争议事实标注争议程度

```
【操作模板】
要件E-02"被告存在违约行为"的待证事实：
  F-02-1：被告未在约定期限（2024年6月30日）前交付货物 [有争议-高]
  F-02-2：被告交付的货物质量不符合合同约定标准 [有争议-中]
  F-02-3：被告未经原告同意擅自变更交付地点 [无争议]
```

### 阶段二：证据盘点与分类

**目标：** 全面盘点现有证据材料，进行系统分类。

#### 步骤4：证据清单编制

- 逐一登记所有证据材料
- 标注证据的基本属性（类型、来源、形式、原件/复印件）
- 初步评估证据的真实性、合法性、关联性（三性）

```
【证据登记模板】
┌──────┬────────┬──────┬──────┬──────┬──────────┐
│证据编号│ 证据名称  │证据类型│证据来源│ 形式  │ 三性初评   │
├──────┼────────┼──────┼──────┼──────┼──────────┤
│ P-01  │采购合同   │ 书证  │ 双方  │ 原件  │真✓ 合✓ 关✓│
│ P-02  │送货单    │ 书证  │ 原告  │ 复印件│真? 合✓ 关✓│
│ P-03  │质检报告   │ 书证  │第三方 │ 原件  │真✓ 合✓ 关✓│
│ P-04  │微信聊天记录│电子数据│ 原告  │ 截图  │真? 合? 关✓│
│ P-05  │证人张某证言│证人证言│ 原告方│ 书面  │真? 合✓ 关✓│
└──────┴────────┴──────┴──────┴──────┴──────────┘
```

#### 步骤5：证据分组

按照证据的证明方向进行初步分组：

- **正向证据**：支持己方主张的证据
- **反向证据**：可能被对方利用的不利证据
- **中性证据**：证明背景事实的证据
- **补强证据**：用于增强其他证据证明力的证据

### 阶段三：证据-主张挂钩（核心步骤）

**目标：** 建立证据与构成要件之间的精确对应关系。

#### 步骤6：逐要件匹配证据

这是整个流程的核心操作。对每个构成要件，逐一检索可用证据：

```
【挂钩操作规则】
1. 一项证据可以同时挂钩多个要件（一证多用）
2. 一个要件应尽量有多项证据支撑（多证一要件）
3. 挂钩时必须明确该证据对该要件的具体证明目的
4. 标注证据对该要件的证明方式（直接证明/间接证明/补强）
```

```
【证据-要件挂钩矩阵】

              │ E-01    │ E-02    │ E-03    │ E-04
              │合同成立  │违约行为  │违约金条款│金额合理
──────────────┼─────────┼─────────┼─────────┼────────
P-01 采购合同  │ ★直接   │ ○间接   │ ★直接   │ ○间接
P-02 送货单    │         │ ★直接   │         │
P-03 质检报告  │         │ ★直接   │         │
P-04 微信记录  │ ○补强   │ ★直接   │         │
P-05 证人证言  │         │ ○补强   │         │

图例：★直接证明  ○间接证明/补强  空白=无关联
```

#### 步骤7：编写证明目的说明

为每个挂钩关系撰写具体的证明目的：

```
【证明目的说明模板】
证据P-04（微信聊天记录）→ 要件E-02（违约行为）
  证明目的：证明被告于2024年7月5日在微信中承认"货物确实还没发出"，
           直接证明被告未在约定期限前交付货物的事实。
  证明方式：直接证明
  关键内容：2024年7月5日14:23，被告法定代表人李某发送消息
           "抱歉，工厂那边出了点问题，货确实还没发出"
  证明强度：强（当事人自认）
```

#### 步骤8：构建证据链

当单一证据不足以充分证明某一要件时，需要构建证据链：

```
【证据链构建模板】
要件E-02（被告未按期交付货物）的证据链：

  P-01（采购合同）──→ 确定交付期限为2024年6月30日
        │
        ↓
  P-02（送货单）──→ 显示实际发货日期为2024年7月15日
        │
        ↓
  P-04（微信记录）──→ 被告7月5日承认尚未发货
        │
        ↓
  P-05（证人证言）──→ 仓库管理员证实7月15日才收到货物

  链条逻辑：合同约定期限(P-01) + 被告自认逾期(P-04) 
           + 实际发货记录(P-02) + 证人印证(P-05)
           = 充分证明被告逾期交货15天

  证据链完整性评估：★★★★★（完整，多证据相互印证）
```

### 阶段四：证明力评估与缺口分析

**目标：** 评估每个要件的证据支撑强度，识别薄弱环节。

#### 步骤9：逐要件证明力评估

```
【证明力评估标准】

等级    │ 标准                              │ 说明
────────┼───────────────────────────────────┼──────────────
★★★★★ │ 多项直接证据相互印证，无矛盾         │ 证明力极强
★★★★☆ │ 有直接证据+补强证据，基本无矛盾      │ 证明力强
★★★☆☆ │ 有直接证据但缺乏补强，或仅有间接证据链 │ 证明力中等
★★☆☆☆ │ 仅有单一间接证据或证据存在瑕疵        │ 证明力较弱
★☆☆☆☆ │ 几乎无证据或证据严重不足             │ 证明力极弱
```

```
【要件证明力汇总表】
┌──────┬──────────────┬──────────┬──────┬──────────────┐
│要件编号│    要件内容     │ 支撑证据数 │证明力 │    风险提示     │
├──────┼──────────────┼──────────┼──────┼──────────────┤
│ E-01  │ 合同有效成立    │    2     │★★★★★│ 无明显风险      │
│ E-02  │ 被告违约行为    │    4     │★★★★★│ 送货单为复印件   │
│ E-03  │ 违约金条款有效   │    1     │★★★★☆│ 需确认格式条款问题│
│ E-04  │ 违约金数额合理   │    1     │★★☆☆☆│ ⚠ 证据薄弱     │
└──────┴──────────────┴──────────┴──────┴──────────────┘
```

#### 步骤10：证据缺口识别与补救建议

```
【证据缺口分析模板】
┌──────────────────────────────────────────────┐
│ ⚠ 证据缺口警告                                │
├──────────────────────────────────────────────┤
│ 缺口位置：要件E-04（违约金数额合理）              │
│ 缺口描述：缺乏证明实际损失金额的直接证据           │
│ 风险等级：高（被告可能申请调减违约金）              │
│ 补救方案：                                     │

│ 优先级：方案A > 方案B > 方案C > 方案D             │
│ 时限提醒：举证期限截止前完成补充                   │
└──────────────────────────────────────────────┘
```

### 阶段五：论证链整合与优化

**目标：** 将分散的证据-要件对应关系整合为完整的论证链，并进行优化。

#### 步骤11：论证链全景图绘制

将所有主张、要件、证据整合为一张完整的论证链全景图（见输出格式模板部分）。

#### 步骤12：论证顺序优化

```
【论证顺序优化原则】
1. 先强后弱：将证据最充分的要件放在前面，建立可信度
2. 先简后繁：简单明了的事实先行，复杂争议后置
3. 时间线索：按事实发生的时间顺序组织，便于理解
4. 逻辑递进：按照法律逻辑的递进关系排列
5. 攻防兼顾：预判对方可能的反驳，提前布置应对证据
```

#### 步骤13：预判对方反驳并准备应对

```
【攻防预判模板】
┌──────────┬──────────────┬──────────────┬──────────┐
│ 己方主张   │ 对方可能反驳    │ 应对证据/策略   │ 应对强度  │
├──────────┼──────────────┼──────────────┼──────────┤
│被告逾期交货│不可抗力（疫情） │P-04微信记录显示 │  强      │
│          │              │系工厂自身问题   │          │
├──────────┼──────────────┼──────────────┼──────────┤
│违约金50万 │金额过高应调减   │补充实际损失证据  │  中（待补）│
└──────────┴──────────────┴──────────────┴──────────┘
```

## 三、常见领域与法源对照

### 3.1 不同案件类型的要件分解参考

| 案件类型 | 典型主张 | 核心构成要件 | 主要法源 |
|---------|---------|------------|---------|
| **合同违约** | 要求承担违约责任 | ①合同成立有效 ②违约事实 ③损害结果 ④因果关系 | 《民法典》第577-585条 |
| **侵权责任（一般）** | 要求赔偿损失 | ①侵权行为 ②损害后果 ③因果关系 ④过错 | 《民法典》第1165条 |
| **侵权责任（特殊）** | 要求赔偿损失 | ①侵权行为 ②损害后果 ③因果关系（举证倒置） | 《民法典》第1169-1178条等 |
| **劳动争议** | 要求支付经济补偿 | ①劳动关系存在 ②解除/终止事由 ③补偿计算基数 ④工作年限 | 《劳动合同法》第46-47条 |
| **知识产权** | 要求停止侵权 | ①权利有效存在 ②被告实施了侵权行为 ③未经授权 | 《著作权法》《专利法》《商标法》 |
| **民间借贷** | 要求偿还借款 | ①借贷合意 ②款项交付 ③还款期限届满 ④未偿还 | 《民法典》第667-680条 |
| **婚姻家事** | 要求分割财产 | ①婚姻关系存在 ②财产属于夫妻共同财产 ③分割方案合理 | 《民法典》第1062-1066条 |
| **公司纠纷** | 股东损害公司利益 | ①股东身份 ②损害行为 ③公司利益受损 ④因果关系 | 《公司法》第20-21条 |

### 3.2 举证责任分配的法源依据

| 规则 | 法源 | 适用场景 |
|------|------|---------|
| 谁主张谁举证 | 《民事诉讼法》第67条 | 一般原则 |
| 举证责任倒置 | 相关司法解释 | 医疗损害、环境污染、产品责任等 |
| 举证妨碍推定 | 《民诉法解释》第112条 | 一方控制证据拒不提交 |
| 自认规则 | 《民事证据规定》第3-5条 | 当事人对不利事实的承认 |
| 书证提出命令 | 《民事证据规定》第45-48条 | 书证在对方控制之下 |
| 证明标准 | 《民事证据规定》第86条 | 高度盖然性标准 |

## 四、验证与筛选规则

### 4.1 证据挂钩有效性验证

每个证据-要件挂钩关系必须通过以下五项验证：

| 验证项 | 验证内容 | 通过标准 | 未通过处理 |
|--------|---------|---------|-----------|
| **关联性验证** | 该证据与该要件是否存在逻辑关联 | 能够合理推导出待证事实 | 移除挂钩关系 |
| **证明方向验证** | 该证据是否支持（而非反驳）该要件 | 证明方向一致 | 标记为不利证据并预备应对 |
| **证明目的明确性** | 能否用一句话说清该证据证明什么 | 证明目的具体、明确 | 重新撰写证明目的 |
| **证据三性验证** | 真实性、合法性、关联性是否满足 | 三性均无明显瑕疵 | 标记风险并准备补强方案 |
| **非重复性验证** | 是否与其他已挂钩证据完全重复 | 提供增量证明价值 | 标记为补强证据或移除 |

### 4.2 证据链完整性筛选规则

```
【完整性检查规则】

规则1：每个积极要件至少有1项直接证据
  → 不满足：标记为"证据缺口-严重"

规则2：关键要件（争议焦点对应的要件）至少有2项以上证据
  → 不满足：标记为"证据缺口-中等"

规则3：每项直接证据至少有1项补强证据
  → 不满足：标记为"证据薄弱-需补强"

规则4：证据链中不存在逻辑断裂
  → 不满足：标记为"逻辑断裂-需修补"

规则5：不存在未被挂钩的"孤立证据"
  → 不满足：重新审视该证据的证明价值或移除
```

### 4.3 论证链逻辑验证

```
【逻辑验证清单】
□ 主张与法律依据是否匹配（请求权基础是否正确）
□ 要件分解是否完整（是否遗漏必要要件）
□ 待证事实是否涵盖要件的全部内涵
□ 证据是否能够证明待证事实（而非其他事实）
□ 证据之间是否存在矛盾（己方证据自相矛盾）
□ 论证链是否存在循环论证
□ 推理过程是否存在跳跃（缺少中间环节）
□ 举证责任分配是否正确
```

## 六、置信度标注体系

### 6.1 整体分析置信度

| 置信度等级 | 标记 | 含义 | 适用条件 |
|-----------|------|------|---------|
| **高置信度** | 🟢 | 分析结论可靠性高 | 案件事实清晰、证据充分、法律关系明确 |
| **中置信度** | 🟡 | 分析结论有一定参考价值但存在不确定性 | 部分事实不清、证据有瑕疵、法律适用有争议 |
| **低置信度** | 🔴 | 分析结论仅供参考，需专业人士进一步判断 | 事实严重不清、关键证据缺失、法律问题复杂 |

### 6.2 单项挂钩置信度

对每个证据-要件挂钩关系标注置信度：

```
【置信度标注格式】
P-01 → E-02 [置信度: 高🟢]
  理由：合同原件直接载明交付期限，证据真实性无争议

P-04 → E-02 [置信度: 中🟡]
  理由：微信记录截图未经公证，对方可能质疑真实性
  建议：建议进行公证保全或区块链存证

P-05 → E-02 [置信度: 低🔴]
  理由：证人为原告公司员工，存在利害关系，证明力可能被削弱
  建议：寻找无利害关系的第三方证人
```

### 6.3 影响置信度的因素

```
提升置信度的因素：
  + 证据为原件
  + 证据经过公证
  + 多项证据相互印证
  + 对方当事人自认
  + 国家机关出具的文书
  + 法律关系简单明确
  + 有明确的司法解释或指导案例

降低置信度的因素：
  - 证据为复印件/截图
  - 证据来源单一
  - 证人与当事人有利害关系
  - 电子数据未经保全
  - 法律适用存在争议
  - 事实认定依赖推理而非直接证据
  - 存在相反证据
  - 案件涉及新型法律问题
```

## 七、常见错误与防范

### 7.1 致命错误表

| 编号 | 错误类型 | 错误描述 | 后果 | 防范措施 |
|------|---------|---------|------|---------|
| **F-01** | 要件遗漏 | 未完整分解法律构成要件，遗漏关键要件 | 整个主张因缺少必要要件的证据而败诉 | 严格对照法条逐一提取要件；参考权威教材的要件分解；交叉验证 |
| **F-02** | 举证责任错配 | 将应由对方承担的举证责任误认为己方承担，或反之 | 浪费举证资源或遗漏必要举证 | 查阅举证责任分配的法律规定和司法解释；区分积极要件与消极要件 |
| **F-03** | 证据-要件错挂 | 将证据挂钩到不相关的要件上 | 关键要件实际无证据支撑而未被发现 | 逐一验证每个挂钩的关联性；撰写明确的证明目的 |
| **F-04** | 忽视不利证据 | 只关注有利证据，忽略己方证据中的不利内容 | 庭审中被对方利用，措手不及 | 对每项证据进行正反两面分析；模拟对方视角审视证据 |
| **F-05** | 证据链断裂 | 证据之间缺乏逻辑衔接，无法形成完整证明 | 法官无法形成心证，达不到证明标准 | 绘制证据链图示，检查每个环节的逻辑衔接 |
| **F-06** | 证明标准误判 | 对证明标准的要求判断错误（如将刑事标准用于民事案件） | 过度举证浪费资源或举证不足 | 明确案件适用的证明标准（民事：高度盖然性） |

### 7.2 常见陷阱

| 编号 | 陷阱名称 | 描述 | 识别方法 | 应对策略 |
|------|---------|------|---------|---------|
| **T-01** | 证据堆砌陷阱 | 大量提交证据但未明确证明目的，导致法官无法抓住重点 | 检查是否每项证据都有明确的证明目的说明 | 精选核心证据，明确标注证明目的，按逻辑顺序排列 |
| **T-02** | 孤证依赖陷阱 | 关键要件仅依赖单一证据，一旦该证据被排除则全盘崩溃 | 检查关键要件是否有备选证据 | 为关键要件准备至少2条独立的证据路径 |
| **T-03** | 自相矛盾陷阱 | 己方不同证据之间存在矛盾，削弱整体可信度 | 交叉比对所有证据的时间、金额、人物等关键信息 | 发现矛盾后及时解释或调整证据组合 |
| **T-04** | 过度推理陷阱 | 从证据到待证事实的推理跳跃过大，缺乏中间环节 | 检查每步推理是否有充分依据 | 补充中间环节的证据，或降低推理的跳跃幅度 |
| **T-05** | 形式瑕疵陷阱 | 证据内容充分但形式不合要求（如未经公证、无原件） | 逐一检查证据形式要求 | 提前进行公证保全、准备原件、完善证据形式 |
| **T-06** | 时间线混乱陷阱 | 证据呈现的时间线不清晰或存在矛盾 | 按时间顺序排列所有证据，检查时间逻辑 | 制作案件时间线，确保证据时间线连贯一致 |
| **T-07** | 忽视程序证据 | 只关注实体证据，忽略程序性事项的证据（如送达、通知） | 检查是否有程序性要件需要证明 | 将程序性要件纳入要件分解 |

## 八、特殊场景处理

### 8.1 多主张竞合场景

当存在多个可选择的请求权基础时（如违约责任与侵权责任竞合）：

```
处理方法：
1. 分别为每个请求权基础构建独立的证据论证链
2. 比较各论证链的证据充分度
3. 标注共用证据（可同时支撑多个主张的证据）
4. 建议选择证据最充分的请求权基础作为主要主张
5. 将其他请求权基础作为备位主张

【竞合分析模板】
┌─────────────┬──────────────┬──────────────┐
│              │ 违约责任路径    │ 侵权责任路径    │
├─────────────┼──────────────┼──────────────┤
│ 要件数量      │     4个       │     4个        │
│ 已有证据覆盖率 │    100%       │     75%        │
│ 最弱要件      │ 损失金额(★★★) │ 过错证明(★★)   │
│ 举证难度      │     中        │      高        │
│ 赔偿范围      │ 可预见性限制   │ 完全赔偿       │
│ 建议         │ ★主要主张     │  备位主张       │
└─────────────┴──────────────┴──────────────┘
```

### 8.2 举证责任倒置场景

```
处理方法：
1. 明确识别哪些要件适用举证责任倒置
2. 对倒置的要件，分析对方可能提交的反证
3. 准备针对对方反证的再反驳证据
4. 注意：即使举证责任倒置，己方仍需提供初步证据

【倒置场景示例：医疗损害责任】
  患方需证明：①就诊事实 ②损害后果
  医方需证明：③诊疗行为无过错 ④诊疗行为与损害无因果关系
  
  → 患方证据链重点：就诊记录、损害鉴定
  → 预判医方反证：病历、诊疗规范、专家意见
  → 准备再反驳：病历真实性质疑、第三方鉴定
```

### 8.3 电子证据场景

```
处理方法：
1. 电子数据需特别关注真实性证明
2. 建议进行公证保全或区块链存证
3. 注意电子数据的完整性（不能截取片段）
4. 准备电子数据的生成、存储、传输过程说明

【电子证据补强清单】
□ 是否进行了公证保全
□ 是否有哈希值校验
□ 是否保留了完整的上下文
□ 是否能证明电子数据的生成主体
□ 是否有其他证据印证电子数据内容
□ 是否符合《电子签名法》等相关规定
```

### 8.4 证据灭失/难以取得场景

```
处理方法：
1. 申请法院调查取证（《民事诉讼法》第67条第2款）
2. 申请书证提出命令（《民事证据规定》第45条）
3. 主张举证妨碍推定（《民诉法解释》第112条）
4. 通过间接证据链替代直接证据
5. 利用经验法则和事实推定

【替代证明路径模板】
原始证据：[已灭失/无法取得的证据]
替代方案：
  路径A：申请法院调取 → 可行性[高/中/低] → 预计时间[X天]
  路径B：间接证据链 → 所需证据[列举] → 证明力[评估]
  路径C：举证妨碍推定 → 前提条件[是否满足] → 成功率[评估]
  路径D：事实推定 → 推定基础[说明] → 对方反驳可能性[评估]
```

### 8.5 群体诉讼/多当事人场景

```
处理方法：
1. 区分共同证据与个别证据
2. 为每个当事人建立独立的证据子链
3. 标注证据的共享关系
4. 注意不同当事人之间的利益冲突对证据使用的影响

【多当事人证据矩阵】
              │ 当事人A │ 当事人B │ 当事人C │ 共用
──────────────┼─────────┼─────────┼─────────┼──────
P-01 合同     │   ✓     │   ✓     │         │  ✓
P-02 付款凭证  │   ✓     │         │         │
P-03 会议纪要  │         │   ✓     │   ✓     │
P-04 鉴定报告  │   ✓     │   ✓     │   ✓     │  ✓
```

## 十、完整示例

### 示例一：简单场景——民间借贷纠纷

#### 案情简介

张三于2023年3月15日向李四借款10万元，约定年利率8%，借期一年，2024年3月15日前还清。张三出具了借条，李四通过银行转账支付了借款。借期届满后，张三未还款。李四起诉要求张三偿还借款本金10万元及利息。

#### 证据论证链分析

**一、主张体系**

```
主张C-01：被告张三应偿还原告李四借款本金10万元
  法律依据：《民法典》第675条
  举证责任：原告
  主张类型：主要主张

主张C-02：被告张三应支付借款利息（按年利率8%计算）
  法律依据：《民法典》第674条、第680条
  举证责任：原告
  主张类型：主要主张
```

**二、要件分解**

```
主张C-01的构成要件：
┌──────┬──────────────────┬──────────┬──────┐
│要件编号│      要件内容       │ 举证责任方 │争议程度│
├──────┼──────────────────┼──────────┼──────┤
│ E-01  │ 借贷合意（双方达成借款合意）│   原告    │  低  │
│ E-02  │ 款项交付（借款已实际交付）  │   原告    │  低  │
│ E-03  │ 还款期限届满              │   原告    │  无  │
│ E-04  │ 被告未偿还借款            │   原告    │  低  │
└──────┴──────────────────┴──────────┴──────┘

主张C-02的构成要件：
┌──────┬──────────────────┬──────────┬──────┐
│要件编号│      要件内容       │ 举证责任方 │争议程度│
├──────┼──────────────────┼──────────┼──────┤
