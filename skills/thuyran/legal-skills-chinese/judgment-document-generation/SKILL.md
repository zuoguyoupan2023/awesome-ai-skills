# 裁判文书生成_8skills

## 一、能力概述

本复合能力面向刑事一审案件，接收案件事实描述（含公诉机关指控、证据概要、被告人供述情况等），输出一份结构完整、论证严密、格式规范的刑事判决书。执行过程严格按照"事实提取→概念理解→争议识别→法条检索→案例检索→演绎推理→格式适用→术语规范"的主线展开，并在关键节点设置质量检查与条件分支。

---

## 二、选用的原子能力清单

| 步骤序号 | 能力编号 | 能力名称 | 选择理由 |
|:---:|:---:|---|---|
| S1 | 4 | 法律核心要素提取 | 裁判文书的基础是准确认定事实，必须从案件描述中提取主体、行为、客体、时间、因果、法律后果等全部核心要素，为后续所有步骤提供结构化输入 |
| S2 | 5 | 法律概念理解 | 需要精准理解涉案罪名的法定构成要件、犯罪形态（既遂/未遂/中止）、共同犯罪、量刑情节等概念的内涵与外延，确保法律适用不偏差 |
| S3 | 6 | 争议焦点和法律关系识别 | 判决书"本院认为"部分必须回应控辩双方争议焦点，需要准确识别案件核心争议点及法律关系性质 |
| S4 | 8 | 法条检索 | 判决书必须援引具体法律条文作为裁判依据，需要检索适用的刑法条文、司法解释及量刑规范 |
| S5 | 9 | 案例检索 | 量刑需要参考同类案件的司法实践规律，确保判决结果在合理区间内，增强裁判的统一性和说服力 |
| S6 | 16 | 演绎推理 | 判决书的核心论证逻辑是"大前提（法律规范）→小前提（案件事实）→结论（裁判结果）"，演绎推理是生成裁判理由和量刑结论的关键能力 |
| S7 | 49 | 法律文书格式适用 | 刑事判决书有严格的格式规范（首部、事实、理由、判决主文、尾部），必须识别文书类型并套用对应格式 |
| S8 | 50 | 法律术语规范表达 | 判决书是正式法律文书，所有措辞必须准确、无歧义、符合法律文体要求，贯穿全文输出过程 |

---

## 三、未选用的原子能力

本复合能力选用了全部8个可用原子能力，无排除项。

**理由说明：** 裁判文书生成是法律AI中最复杂的终端任务之一，它要求从事实认定到法律适用再到文书输出的全链条覆盖。8个原子能力分别对应该链条中不可或缺的环节——事实层（AS4）、概念层（AS5）、争议层（AS6）、规范层（AS8、AS9）、推理层（AS16）、输出层（AS49、AS50）。缺少任何一个都会导致判决书在某个维度上存在缺陷。

### S1：法律核心要素提取

| 项目 | 内容 |
|---|---|
| **调用能力** | AS4 — 法律核心要素提取 |
| **输入** | 原始案件事实描述全文（含公诉机关指控、证据列举、被告人态度） |
| **执行动作** | 从文本中逐项提取以下不可再分的核心要素：①犯罪主体（被告人身份、共犯信息）；②犯罪行为（具体行为方式、手段）；③犯罪客体/对象（侵害的法益、具体财物）；④犯罪目的/动机；⑤犯罪结果（既遂/未遂、损失金额）；⑥时间线（案发时间、归案时间、退赔时间等）；⑦因果关系；⑧量刑相关情节（自首、坦白、退赃、前科等）；⑨证据概要 |
| **输出** | 结构化要素清单（JSON或表格形式） |
| **传递给下一步** | 完整的结构化要素清单，供S2理解相关法律概念 |

**案例一示例：**
> 提取结果：主体（张3）、行为（伪造身份证、冒名骗财）、目的（偿还债务）、目标金额（10万元）、结果（被识破未得手）、时间（2017年4月28日）

**案例二示例：**
> 提取结果：主体（安旭＋杨某共同犯罪）、行为（持工具撬门入户盗窃）、客体（金银首饰）、时间（2016年5月9日案发，2018年4月15日被抓）、从宽情节（退缴3000元）

---

### S2：法律概念理解

| 项目 | 内容 |
|---|---|
| **调用能力** | AS5 — 法律概念理解 |
| **输入** | S1输出的结构化要素清单 |
| **执行动作** | 根据提取的行为特征和结果，识别可能涉及的罪名，并逐一理解以下法律概念的构成要件和内涵外延：①涉案罪名的法定构成要件（四要件或三阶层）；②犯罪形态概念（既遂、未遂、中止、预备）；③共同犯罪概念（主犯、从犯、胁从犯）；④量刑情节概念（法定从轻/减轻/从重情节、酌定情节）；⑤其他涉案特殊概念（如"入户盗窃""数额巨大"等） |
| **输出** | 涉案法律概念解析表（每个概念含定义、构成要件、适用条件） |
| **传递给下一步** | 概念解析表 + S1的要素清单，供S3识别争议焦点 |

**案例一示例：**
> 理解诈骗罪构成要件——以非法占有为目的、虚构事实或隐瞒真相、骗取他人财物、数额较大；犯罪未遂——已着手实行犯罪，因意志以外原因未得逞

**案例二示例：**
> 理解盗窃罪"入户盗窃"——无论金额大小均入罪；共同犯罪；主犯与从犯的区分标准

---

### 🔍 质量检查点 QC1：要素-概念匹配验证

| 检查项 | 验证逻辑 | 不通过处理 |
|---|---|---|
| 要素完整性 | S1提取的要素是否覆盖了S2中罪名构成要件的全部对应事实 | 回溯S1，补充提取遗漏要素 |
| 概念覆盖性 | S2是否覆盖了S1中所有行为特征对应的法律概念 | 在S2中补充理解遗漏概念 |
| 罪名初判合理性 | 初步识别的罪名是否与行为特征匹配（如暴力取财不应定盗窃） | 回溯S2重新分析罪名方向 |

---

### S3：争议焦点和法律关系识别

| 项目 | 内容 |
|---|---|
| **调用能力** | AS6 — 争议焦点和法律关系识别 |
| **输入** | S1的结构化要素清单 + S2的法律概念解析表 |
| **执行动作** | ①识别当事人之间的法律关系性质（刑事追诉关系、共犯关系、被害人与被告人关系等）；②识别控辩双方可能存在的争议焦点，包括但不限于：罪名认定争议、犯罪形态争议（既遂vs未遂）、共犯地位争议（主犯vs从犯）、量刑情节争议（自首/坦白认定）、数额认定争议；③对每个争议焦点标注需要在"本院认为"部分回应的优先级 |
| **输出** | 争议焦点清单（含优先级排序）+ 法律关系图谱 |
| **传递给下一步** | 争议焦点清单，指导S4精准检索相关法条 |

**案例一示例：**
> 争议焦点：①是否构成诈骗罪既遂或未遂；②"数额巨大"认定（10万元适用第266条哪一量刑档次）

**案例二示例：**
> 争议焦点：①安旭在共同犯罪中的地位（主犯还是从犯）；②退缴3000元对量刑的影响

---

### S4：法条检索

| 项目 | 内容 |
|---|---|
| **调用能力** | AS8 — 法条检索 |
| **输入** | S2的罪名及概念解析 + S3的争议焦点清单 |
| **执行动作** | 按以下层次检索适用法律规范：①**定罪法条**：涉案罪名对应的刑法分则条文（含量刑幅度）；②**犯罪形态法条**：如未遂（第23条）、中止（第24条）等总则条文；③**共犯法条**：如共同犯罪（第25条）、主从犯（第26-29条）；④**量刑情节法条**：如自首（第67条）、坦白（第67条第三款）、立功（第68条）等；⑤**刑罚执行法条**：罚金（第52条、第53条）、退赔（第64条）等；⑥**司法解释**：涉案罪名的数额标准、量刑指导意见等 |
| **输出** | 适用法条清单（含条文全文、适用理由、对应争议焦点编号） |
| **传递给下一步** | 法条清单，供S5案例检索时限定检索范围，供S6作为演绎推理的大前提 |

**案例一示例：**
> 检索结果：《刑法》第266条（诈骗罪及量刑幅度）、第23条（犯罪未遂）、第67条第三款（如实供述从轻）

**案例二示例：**
> 检索结果：《刑法》第264条、第25条、第67条第三款、第52条、第53条、第64条

---

### 🔍 质量检查点 QC2：法条-争议覆盖验证

| 检查项 | 验证逻辑 | 不通过处理 |
|---|---|---|
| 争议焦点全覆盖 | S3中每个争议焦点是否都有对应的法条支撑 | 回溯S4补充检索 |
| 罪名条文准确性 | 检索到的罪名条文是否与S2中确定的罪名一致 | 回溯S2重新确认罪名，或回溯S4修正检索 |
| 量刑情节法条完整性 | S1中提取的每个量刑情节是否都有对应法条 | 回溯S4补充检索量刑情节相关法条 |
| 司法解释时效性 | 检索到的司法解释是否在案发时有效 | 替换为案发时有效的司法解释版本 |

### S5：案例检索

| 项目 | 内容 |
|---|---|
| **调用能力** | AS9 — 案例检索 |
| **输入** | S1的结构化要素（罪名、行为方式、金额、情节）+ S3的争议焦点 + S4的法条清单 |
| **执行动作** | ①以罪名＋行为方式＋犯罪形态为关键词，检索同类案件判决；②重点关注：相似金额/情节下的量刑区间、争议焦点的裁判倾向（如从犯认定标准的实践把握）、同一地区法院的量刑习惯；③提取类案的量刑基准和调节因素 |
| **输出** | 类案检索报告（含3-5个参考案例的罪名、情节、量刑结果、裁判要旨） |
| **传递给下一步** | 类案量刑区间和裁判倾向，供S6演绎推理时校准量刑结论 |

**条件分支：**
- 若检索到高度相似案例（行为方式、金额、情节基本一致）：直接提取量刑参考值
- 若未检索到高度相似案例：扩大检索范围（放宽金额区间或行为方式），并标注参考价值有限

**案例一示例：**
> 查找同类诈骗未遂案件的量刑规律，辅助判断七个月量刑区间的合理性

**案例二示例：**
> 从同类入户盗窃案件归纳上海地区司法实践量刑规律；查找从犯认定标准的相关判例

---

### S6：演绎推理

| 项目 | 内容 |
|---|---|
| **调用能力** | AS16 — 演绎推理 |
| **输入** | S1的结构化要素（小前提）+ S4的法条清单（大前提）+ S3的争议焦点（论证目标）+ S5的类案参考（量刑校准） |
| **执行动作** | 按以下逻辑链逐层推理，每一层均采用"大前提→小前提→结论"的三段论结构：**第一层：定罪推理** — 将案件事实逐项对照罪名构成要件，论证是否满足全部要件→得出罪名成立/不成立的结论；**第二层：犯罪形态推理**（条件触发）— 若存在未遂/中止等形态争议，将事实对照形态要件→得出形态结论；**第三层：共犯推理**（条件触发）— 若存在共同犯罪，论证各被告人的地位和作用→得出主从犯结论；**第四层：量刑情节推理** — 逐项论证法定/酌定从轻、减轻、从重情节的成立与否；**第五层：量刑结论推理** — 综合定罪结论＋形态＋共犯地位＋量刑情节＋类案参考→推导具体刑期和附加刑；**第六层：附带事项推理** — 退赔、没收、禁止令等附带判项的推导 |
| **输出** | 完整推理链（含每层的大前提、小前提、结论）+ 最终裁判结论（主刑＋附加刑＋附带判项） |
| **传递给下一步** | 推理链文本（供"本院认为"部分使用）+ 裁判结论（供判决主文使用） |

**案例一示例：**
> ①诈骗罪构成要件逐项满足→构成诈骗罪；②被识破未得手属意志以外原因→犯罪未遂成立；③10万元属"数额巨大"量刑档，未遂可比照既遂从轻或减轻；④叠加如实供述从轻→推导至七个月有期徒刑＋罚金三万元

**案例二示例：**
> ①入户盗窃构成要件满足→盗窃罪成立；②二人共谋共同实施→共同犯罪成立；③辩护人主张从犯，无充分证据支撑→不予采纳；④如实供述＋退缴从轻→推导六个月有期徒刑＋罚金二千元＋退赔指令

---

### 🔍 质量检查点 QC3：推理链完整性与一致性验证

| 检查项 | 验证逻辑 | 不通过处理 |
|---|---|---|
| 构成要件全覆盖 | 定罪推理是否逐项论证了罪名的每个构成要件 | 回溯S6第一层，补充遗漏要件的论证 |
| 争议焦点全回应 | S3中每个争议焦点是否在推理链中得到明确回应 | 回溯S6，补充对遗漏争议焦点的论证 |
| 量刑合理性 | 推导出的量刑是否落在S4法条规定的量刑幅度内，且与S5类案量刑区间基本一致 | 若超出法定幅度→回溯S4确认法条适用是否正确；若与类案偏差过大→回溯S5扩大检索或在S6中补充特殊情节说理 |
| 逻辑自洽性 | 各层推理结论之间是否存在矛盾（如认定未遂却按既遂量刑） | 回溯S6修正矛盾环节 |

### S7：法律文书格式适用

| 项目 | 内容 |
|---|---|
| **调用能力** | AS49 — 法律文书格式适用 |
| **输入** | S1的结构化要素 + S6的推理链和裁判结论 + S4的法条清单 |
| **执行动作** | ①识别文书类型为"刑事一审判决书"；②按照最高人民法院规定的刑事判决书格式规范，将前序步骤的输出组装为以下结构：**首部** — 法院名称、案号、公诉机关、被告人基本信息、辩护人信息、案件由来和审理经过；**事实部分** — "经审理查明"段（基于S1要素重述事实）＋证据列举；**理由部分** — "本院认为"段（基于S6推理链展开论证，逐一回应争议焦点）；**判决主文** — 定罪量刑结论＋附加刑＋退赔等附带判项（基于S6裁判结论）；**尾部** — 刑期折抵计算、上诉权告知、审判人员署名、日期；**附录** — 相关法律条文全文（基于S4法条清单） |
| **输出** | 格式化判决书草稿（含全部结构板块） |
| **传递给下一步** | 判决书草稿全文，供S8进行术语规范化润色 |

**案例一示例：**
> 刑事判决书格式，依次写明当事人信息、公诉机关指控、事实认定、本院认为、判决主文、刑期计算、上诉告知、附相关法律条文

---

### S8：法律术语规范表达

| 项目 | 内容 |
|---|---|
| **调用能力** | AS50 — 法律术语规范表达 |
| **输入** | S7输出的判决书草稿全文 |
| **执行动作** | 对判决书全文进行逐句审查和规范化处理：①**术语准确性**：确保罪名表述、犯罪形态表述、量刑情节表述使用法定术语（如"犯罪未遂"而非"未遂犯"，"如实供述自己的罪行"而非"坦白交代"）；②**判决主文规范**：确保"判处有期徒刑X年/月""并处罚金人民币X元""责令退赔"等表述符合司法文书惯例；③**法条引用规范**：确保"依照《中华人民共和国刑法》第X条第X款"的引用格式正确；④**文体一致性**：确保全文使用第三人称叙述、法律文书专用句式、无口语化表达；⑤**逻辑连接词规范**：确保"鉴于""综上""据此"等连接词使用得当 |
| **输出** | 术语规范化后的判决书终稿 |
| **传递给最终输出** | 完整刑事判决书 |

**案例一示例：**
> 确保"犯罪未遂""比照既遂犯从轻或者减轻处罚""判处有期徒刑""并处罚金"等表述符合刑事文书规范

## 五、条件分支与回溯机制

### 5.1 犯罪形态分支

```
在S1提取结果中检查"犯罪结果"要素：
├── 结果为"未得手/未得逞/未遂" → S2中必须理解"犯罪未遂"概念
│                                  → S3中必须将"既遂vs未遂"列为争议焦点
│                                  → S4中必须检索《刑法》第23条
│                                  → S6中必须增加"犯罪形态推理"层
├── 结果为"中止" → 类似路径，替换为第24条
└── 结果为"既遂/得手" → 跳过犯罪形态相关步骤
```

**案例一触发此分支：** 张3"被识破未得手"→触发未遂路径→S2理解犯罪未遂→S4检索第23条→S6论证未遂成立并适用从轻/减轻

**案例二未触发此分支：** 安旭"窃得金银首饰后逃逸"→既遂→跳过犯罪形态步骤

### 5.2 共同犯罪分支

```
在S1提取结果中检查"犯罪主体"要素：
├── 主体为多人 → S2中必须理解"共同犯罪""主犯/从犯"概念
│               → S3中必须将"各被告人地位"列为争议焦点
│               → S4中必须检索《刑法》第25-29条
│               → S6中必须增加"共犯推理"层
└── 主体为单人 → 跳过共同犯罪相关步骤
```

**案例二触发此分支：** 安旭"伙同杨某"→触发共犯路径→S2理解共同犯罪→S3识别主从犯争议→S6论证不予采纳从犯辩护意见

**案例一未触发此分支：** 张3单独作案→跳过共犯步骤

### 5.3 辩护意见回应分支

```
在S1提取结果中检查是否存在辩护人意见：
├── 存在辩护意见 → S3中必须将辩护意见涉及的问题列为争议焦点
│                 → S6中必须对每条辩护意见逐一回应（采纳/不予采纳并说理）
│                 → S7中"本院认为"部分必须包含对辩护意见的评述
└── 不存在辩护意见 → 跳过辩护意见回应，仅回应控方指控
```

### 5.4 量刑异常回溯

```
在QC3中检查量刑合理性：
├── 量刑超出法定幅度 → 回溯S4确认量刑档次是否正确
│                     → 回溯S6重新计算量刑
├── 量刑与类案偏差>50% → 回溯S5扩大案例检索范围
│                       → 回溯S6补充特殊情节说理
└── 量刑在合理区间 → 继续S7
```

### 5.5 罪名变更回溯

```
在S6定罪推理中发现构成要件不满足：
├── 某一要件明显不满足 → 回溯S2重新分析可能的罪名
│                       → 回溯S3重新识别争议焦点
│                       → 回溯S4重新检索法条
│                       → 从S6重新开始推理
└── 全部要件满足 → 继续后续推理层
```

## 七、异常处理与兜底机制

| 异常场景 | 处理方式 |
|---|---|
| 案件事实描述过于简略，S1无法提取足够要素 | 在输出中标注"事实不清"，列出缺失要素清单，请求补充信息后重新执行 |
| S2无法确定唯一罪名（存在罪名竞合） | 在S3中将罪名选择列为首要争议焦点，在S6中对每个候选罪名分别进行构成要件论证，择一适用并说理 |
| S5未检索到任何类案 | 标注"无直接类案参考"，S6量刑推理仅依据法条规定的量刑幅度，并在判决书中加强说理论证 |
| S6推理链出现逻辑矛盾 | 触发全链回溯，从S1重新审查要素提取是否准确，逐步排查矛盾来源 |
| 案件涉及多罪名数罪并罚 | 对每个罪名分别执行S2-S6的完整流程，在S6末尾增加"数罪并罚推理"层，在S7中按多罪名判决书格式组装 |

## 八、能力调用依赖关系总结

```
AS4 ──→ AS5 ──→ AS6 ──→ AS8 ──→ AS9 ──→ AS16 ──→ AS49 ──→ AS50
 │        │        │        │        │        │         │         │
 │        │        │        │        │        │         │         └─ 终稿输出
 │        │        │        │        │        │         └─ 格式组装
 │        │        │        │        │        └─ 核心推理（依赖AS4+AS8+AS6+AS9）
 │        │        │        │        └─ 量刑校准
 │        │        │        └─ 大前提供给
 │        │        └─ 论证目标设定
 │        └─ 概念框架建立
 └─ 事实基础奠定
```

**关键依赖说明：**
- AS16（演绎推理）是整个流程的核心枢纽，它同时依赖AS4（小前提）、AS8（大前提）、AS6（论证目标）、AS9（量刑校准）四个上游能力的输出
- AS49（格式适用）和AS50（术语规范）是串行的输出层能力，AS49负责结构，AS50负责措辞，二者共同保证最终文书的形式质量
- AS5（概念理解）虽然位于流程早期，但其输出贯穿后续所有步骤，是保证法律适用准确性的基础能力
