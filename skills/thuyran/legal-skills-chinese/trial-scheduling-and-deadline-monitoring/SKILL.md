# 庭审排期与程序时限监控

> 跟踪法院排期、证据提交期限、上诉期限、送达期限，避免程序性风险。

## 触发条件

包括但不限于以下情形：

- 提到“上诉期限”“证据提交期限”“送达期限”“保全期限”等关键词
- 增加、修改、询问案件期限时

若用户要求交付__程序时限监控一览表、证据提交期限一览表、送达期限一览表、上诉期限一览表、保全期限一览表时__，同样使用此技能。

## 排除情形

__请勿使用__此技能处理以下需求：

- 增减、修改、管理某一案件的关键时间地点一览表
- 管理、询问律师、律师助理、实习生的工作时长
- 与案件期限排期无关的一般性编程任务

# Skill Title

跟踪法院排期、证据提交期限、上诉期限、送达期限、保全期限等法定期限，避免程序性风险

## Capabilities

- 当用户增删、修改立案时间、传票送达时间、判决书送达时间、保全时间等法定期限的起算时间点时，根据《民事诉讼法》及相关司法解释，计算可确定的法定期限节点
- 对于每一个案件，当用户没有完整输入所有法定期限起算点时，主动询问用户是否遗漏相关信息
- 在法定期限前5天、前3天分别提醒法官、检察官、仲裁员、律师等某个案件的诉讼期限
- 当用户询问某个案件的期限时，向用户提供案件期限一览表
- 当用户询问某个期限，如证据提交期限时，向用户提供证据提交期限一览表，包括所有需要提交证据的案件和对应期限
- 当用户询问近日期限时，向用户提供截止到30日之后的法律期限一览表，包含此期间内所有期限的截止日期、对应案件、对应工作。

## How to Use

当增添、修改一个需要监控期限的案件时：

1. 创建新增案件名称，或通过案件名称选择已有案件
2. 输入已知的时间起算点，如起诉材料送达法院时间、立案时间、裁判文书送达时间、开庭时间、保全时间等
3. 根据《民事诉讼法》及相关司法解释，计算可确定的法定期限节点
4. 向用户展示计算结果，询问用户是否有需要调整的日期
5. 输出经用户确认过的程序监控期限一览表。

当用户需要调取某一法律期限一览表时：

1. 检索所有涉及此种法律期限的案件
2. 以当前日期为节点，只筛选法定期限截止日期在今日之后的案件
3. 检查是否有案件没有该法律期限的起始日期，需要用户进行补充
4. 将2、3筛选后的案件一并输出于某一法律期限一览表。

当用户需要近日法律期限一览表时：

1. 询问用户筛选期限范围
2. 如果用户提供明确期限（如15日）或日期（如3月2日），则以当前日期为节点，筛选截止到用户明确期限或日期内的法律期限一览表
3. 如果用户没有提供明确期限，则以当前日期为节点，筛选截止到30日之后的法律期限一览表

当提醒用户某一案件的某一法定期限即将达到时

1. 判断用户当前谈论案件是哪一个案件，调取该案件程序监控期限一览表
2. 检查当前日期是否到达法定期限截止日前5日、前3日
3. 输出提醒用户的信息，格式为“注意：xx案件将于5日后（xx年xx月xx日）到达xx期限。“

## Example Usage

__场景：给某一案件增加法定期限起始日__

用户输入：“甲公司那个债权纠纷案子，3月2日收到了一审判决书”

SKILL处理流程：

1. 案件名称匹配：某甲公司诉某乙公司债券纠纷案
2. 识别工作任务：为已有案件增加法定期限截止日
3. 检索《民事诉讼法》及相关司法解释相关规定，检索到《民事诉讼法》第171条
4. 请用户确认“该案上诉期限将于3月17日届满“是否有误
5. 经用户确认无误后，将该法定期限截止日期加入该案程序监控期限一览表，并向用户展示更新后的程序监控期限一览表

## Best Practices

1. 如果用户一次性提及多个案件名称和截止日期，首先确认案件名称与日期之间的关联性，分案件依次进行梳理
2. 如果检索到多个相关法条，无法确认法定期限，向用户询问补充信息（如是普通一审程序、简易程序还是小额诉讼程序）
3. 当用户增加非法定期限时，需要向用户确认2次再写入一览表
4. 在用户创建案件时，主动询问用户是否需要在法定期限截止日期5日之前（如10日、15日）设置额外提醒。

## Limitations

1. 本skill不处理案件关键地点的梳理
2. 本skill不处理工作任务的分配
3. 本skill不处理工作费用管理
