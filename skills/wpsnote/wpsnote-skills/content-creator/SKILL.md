# Content Creator - 专业内容创作工作流

本 Skill 提供从风格提取、需求分析、深度研究到最终创作的完整工作流。强制遵循"先分析、再研究、后创作"的顺序，确保输出质量。

## 边界说明（重要）

本 Skill **只用于从0到1的内容创作**，以下情况**不要使用本 Skill**：

| 场景 | 处理方式 |
|------|----------|
| 已有内容需要排版发布 | 这是发布阶段，不是创作阶段 |
| 简单文本编辑、格式调整 | 直接用 MCP 工具 |
| 翻译、摘要、改写已有内容 | 直接用 Claude 内置能力 |
| 查找已有笔记内容 | 用笔记搜索工具 |

**一句话判断**：如果用户说的是"帮我写...""我想写..."，用本 Skill；如果用户说的是"排版...""发布...""导出..."，说明内容已完成，不需要创作。

## Contract（契约）

### Input Contract

**触发条件**：
- "帮我写篇文章"
- "写个公众号/小红书/知乎回答"
- "创作关于XX的内容"
- "写个稿子"
- "按照这篇的风格写"
- "分析我的写作风格"
- "提取写作模板"
- "仿照这个风格写"

**输入类型**：
1. **样张文章**（可选）：用于提取写作风格的参考文章
2. **创作主题**：一句话描述想写的内容（必填）
3. **目标平台**：公众号/小红书/知乎/今日头条等（可选，默认公众号）
4. **补充要求**：篇幅、语气、特定要点（可选）

**前置条件**：
- 如要提取风格，样张需 > 500 字且为完整文章
- 创作主题需明确（可引导用户细化）

### Output Contract

**输出内容**：
| 文件 | 说明 |
|------|------|
| `persona.md` | 写作风格画像（如基于样张提取） |
| `brief.md` | 需求文档（目标、受众、核心信息） |
| `outline.md` | 文章大纲（结构、要点、研究需求） |
| `research_notes.md` | 研究笔记（内部素材+外部资料） |
| `draft.md` | 初稿内容 |
| `final.md` | 终稿内容 |

**质量保证**：
- 严格遵循提取的写作风格
- 基于研究资料创作，不是凭空发挥
- 符合目标平台的内容规范

### Error Contract

| 错误场景 | 处理方式 |
|---------|---------|
| 样张太短 | 提示提供更长、更完整的样张（>500字） |
| 主题不清晰 | 引导用户回答：写什么？给谁看？想达到什么目的？ |
| 研究素材不足 | 主动请求用户提供更多背景资料或关键词 |

## 核心工作流

强制6步流程，**不可跳过**：

```
风格画像 → 需求确认 → 大纲设计 → 深度研究 → 文章创作 → 定稿交付
```

### Step 1: 风格画像（Persona）

**目标**：建立或加载用户的写作风格画像。

**执行**：
1. 检查是否已有 `persona.md` 或历史模板
2. 如有样张文章，提取风格特征：
   - 句式偏好（长短句、节奏）
   - 词汇特点（专业度、口语化程度）
   - 结构习惯（开头方式、段落长度）
   - 情感基调（严肃/轻松、理性/感性）
3. 生成 `persona.md` 并向用户确认

> **关键验证点**：没有风格画像不得进入创作阶段

### Step 2: 需求确认（Brief）

**目标**：完整理解写作需求。

**必问问题**：
- 这篇文章的目的是什么？（告知/说服/娱乐）
- 目标读者是谁？
- 核心信息/观点是什么？
- 发布在哪里？（公众号/小红书/知乎...）
- 大概需要多长？
- 截稿时间？

**输出**：`brief.md` 需求文档

### Step 3: 大纲设计（Outline）

**目标**：制定清晰的文章结构。

**内容**：
- 标题（工作标题）
- 开篇方式
- 主体结构（2-5个部分）
- 各部分要点
- 结尾方式
- **研究需求标记**：哪些论点需要论据支撑

**输出**：`outline.md` 大纲文档，需用户确认

### Step 4: 深度研究（Research）

**目标**：收集支撑材料，**此步骤不可跳过**。

**内部研究**（WPS 笔记历史）：
- 使用 `search_notes` 搜索用户历史笔记中的相关素材
- 使用 `search_note_content` 在相关笔记内精确查找关键词
- 查找可复用的案例、数据、表达方式

**外部研究**（网络搜索）：
- 搜索最新行业动态、数据报告
- 查找权威观点、案例支撑
- 了解热点话题和讨论

**输出**：`research_notes.md` 研究笔记

> **关键验证点**：每个主要论点需有2+支撑材料

### Step 5: 文章创作（Creation）

**目标**：基于以上所有准备，撰写初稿。

**创作原则**：
1. **风格一致性**：严格遵循 `persona.md` 的风格画像
2. **结构忠实性**：按照 `outline.md` 框架展开
3. **素材整合**：自然融入 `research_notes.md` 的研究成果
4. **需求契合度**：满足 `brief.md` 中的所有要求

**输出**：`draft.md` 初稿

### Step 6: 定稿交付（Delivery）

**目标**：根据反馈修订，输出终稿。

**执行**：
1. 展示 `draft.md` 给用户
2. 收集反馈并修订
3. 保存 `final.md` 终稿
4. （可选）将文章存入样张库，用于未来风格提取

## 常用编排模式

### 模式 1：基于样张创作（推荐）

当用户提供了参考文章，希望仿照其风格创作：

```
检查 persona.md → 如不存在，分析样张提取风格 → 确认需求 → 设计大纲 → 深度研究 → 创作 → 定稿
```

**关键工具**：
- `wps-read.py` 读取样张文章
- `extract-template.py` 提取风格模板

### 模式 2：全新主题创作

当用户只有主题想法，没有特定风格要求：

```
检查 persona.md → 使用通用风格或加载历史画像 → 需求访谈 → 设计大纲 → 深度研究 → 创作 → 定稿
```

### 模式 3：快速迭代创作

当用户对已有初稿不满意，需要大幅修改：

```
加载已有 brief/outline → 重新确认需求变化 → 调整大纲 → 补充研究 → 重写 → 定稿
```

## Troubleshooting

### 风格提取失败

**现象**：`extract-template.py` 报错或提取结果为空
**原因**：样文章节结构不清晰、内容太短（<500字）、XML 解析错误
**解决**：
1. 检查样张是否为完整文章（有头有尾）
2. 确保样张 > 500 字
3. 手动阅读样张，直接观察风格特征

### 研究素材不足

**现象**：`search_notes` 返回结果为空或无关
**原因**：关键词不准确、笔记库中确实无相关内容
**解决**：
1. 尝试同义词扩展搜索
2. 使用 web search 补充外部资料
3. 主动向用户询问背景资料

### 大纲确认后用户大改需求

**现象**：用户说"不对，我不是这个意思"
**原因**：需求访谈阶段理解偏差
**解决**：
1. 不要直接在原大纲上修改，回到 Step 2 重新确认需求
2. 更新 `brief.md` 记录新的理解
3. 基于新 brief 重新设计大纲

### MCP 工具调用失败

**现象**：`mcp__wpsnote__search_notes` 等工具报错
**原因**：EDITOR_NOT_READY、BLOCK_NOT_FOUND、网络问题
**解决**：
1. 检查 WPS 笔记应用是否正常运行
2. 重新获取 block_id（编辑后 ID 会变化）
3. 参考 `wps-note` SKILL 的 Troubleshooting

### 创作偏离风格

**现象**：用户说"这不像我的风格"
**原因**：persona.md 描述不够具体，或创作时未严格遵循
**解决**：
1. 细化 persona.md，添加更多具体示例
2. 创作时逐段对照 persona.md 检查
3. 关键句使用用户原话风格改写

## 与 WPS 笔记的集成

创作完成后，可直接写入 WPS 笔记：

```bash
# 写入 WPS 笔记
python scripts/wps-write.py --input final.md --title "文章标题" --tags "公众号"
```

**通过 MCP 工具写入**：
```
create_note({ title: "文章标题" }) → note_id
batch_edit({ note_id, operations: [
  { op: "replace", block_id, content: "<h1>标题</h1>..." }
]})
```

## Examples（使用示例）

**示例 1：基于样张创作**
- 用户："按照我这篇文章的风格，写一篇关于AI编程工具的"
- Skill：
  1. 分析样张 → 提取 persona.md
  2. 确认需求 → 目标读者是程序员，目的是介绍工具
  3. 设计大纲 → 介绍、对比、推荐三个部分
  4. 深度研究 → 搜索最新AI编程工具资料
  5. 创作初稿 → 按样张风格写作
  6. 定稿交付

**示例 2：新主题创作**
- 用户："帮我写个公众号文章，讲时间管理"
- Skill：
  1. 检查/创建 persona（如用户无特定样张，使用通用风格）
  2. 确认需求 → 职场人群，实用技巧类
  3. 设计大纲
  4. 深度研究 → 搜索时间管理方法论、案例
  5. 创作初稿
  6. 定稿交付
