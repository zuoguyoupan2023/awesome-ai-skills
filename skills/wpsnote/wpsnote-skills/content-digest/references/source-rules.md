# 来源标识与原文附录规则

## 来源类型判断优先级

**以用户最初提供的输入形式为准，不随后续操作改变。**

- 用户给了 URL → 始终为「URL」，即使被拦截后用户补充粘贴了正文，**仍按 URL 处理**
- 用户给了本地文件路径 → 始终为「本地文件」
- 用户直接粘贴文本且**无任何 URL 或文件路径** → 「纯粘贴文本 / 邮件」
- 用户提供图片（本地或网络）→ 「图片」

---

## 来源标识规则（**写在首行 blockquote，文末不重复**）

来源信息只写在笔记开头的 `<blockquote>` 里，格式如下：

| 来源类型 | blockquote 内容 |
|---------|----------------|
| URL / 公众号 | `来源：[URL 超链接] · 类型：网页 · 日期：YYYY-MM-DD` |
| 本地文件（文本类）| `来源：/绝对/文件/路径.pdf · 类型：PDF · 日期：YYYY-MM-DD` |
| 图片（本地或网络）| `类型：[截图 / 设计图 / 图表] · 日期：YYYY-MM-DD`（无来源字段）|
| 纯粘贴文本 / 邮件 | `来源：手动粘贴 / 邮件主题：XXX · 日期：YYYY-MM-DD` |

---

## 原文附录规则

| 来源类型 | 原文附录 |
|---------|---------|
| URL / 公众号（含抓取失败后用户补充粘贴）| 不附原文，只保留链接 |
| 本地文件（文本类）| 不附原文，只保留文件绝对路径 |
| 图片（本地或网络）| 不写任何附录 |
| 纯粘贴文本 / 邮件（无 URL、无文件路径）| `<hr/>` 分隔线 + 完整原文一字不差逐段写入 |

纯粘贴文本的原文附录格式：

```xml
<hr/>
<h2>原文</h2>
<p>[原文第一段，原样保留，不做任何修改]</p>
<p>[原文第二段]</p>
...
```
