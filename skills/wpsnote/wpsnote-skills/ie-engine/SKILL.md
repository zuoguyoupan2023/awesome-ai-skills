# 灵感引擎：编排与注意力管理

本技能是灵感引擎（Inspiration Engine）的统一入口和编排中心。
它负责串联三个分层技能，运行完整的灵感生成流水线，并管理何时、如何向用户呈现洞见。

## 适用场景

### 被动触发（用户明确请求）

- 用户说"帮我看看过去的笔记里有什么启发"
- 用户说"connect the dots"或"帮我把想法串起来"
- 用户正在做创造性工作，想要从历史笔记中获得灵感
- 用户想全面审视自己某个领域的思考演化

### 主动触发（Agent 判断用户可能受益）

当你观察到以下信号时，应主动考虑运行灵感引擎并提醒用户：

- 用户正在深入探讨某个主题，且该主题可能在历史笔记中有相关内容
- 用户提出了一个想法或假设，历史笔记中可能有支撑或矛盾的证据
- 用户在不同对话中反复提及类似的概念，暗示这是一个长期关注的方向
- 用户正在做决策或规划，历史经验可能提供有价值的参考

主动触发时，不要直接运行完整流水线，而是先用一句话轻量提示用户，由用户决定是否深入。详见下方「主动触发协议」。

## 不适用场景

如果用户只需要单一能力，应引导到对应的分层技能：

- 只需要检索笔记 --> ie-retrieve-memory
- 只需要分析笔记关系 --> ie-connect-dots
- 只需要生成洞见 --> ie-generate-insight
- 创建或编辑笔记 --> 直接使用 wpsnote 工具
- 通用问答 --> 直接回答

## 流水线流程

完整的灵感生成流水线包含三个阶段，按顺序执行：

```
阶段一：记忆检索（ie-retrieve-memory）
  输入：用户的当前上下文/主题/关键词
  输出：相关历史笔记列表
  ↓
阶段二：想法连接（ie-connect-dots）
  输入：阶段一的检索结果
  输出：聚类、连接假设、长期模式
  ↓
阶段三：洞见生成（ie-generate-insight）
  输入：阶段二的推理结果
  输出：洞见、探索建议、演化路径
```

详细的流水线定义和参数配置请阅读 `references/pipeline.md`。

## 编排执行步骤

### 第一步：理解上下文

在启动流水线之前，先明确用户的意图和当前上下文：

1. 确定用户关心的主题或领域
2. 确定用户的期望（全面扫描 vs 聚焦某个方向）
3. 如果信息不足，向用户询问补充信息

### 第二步：运行记忆检索

按照 ie-retrieve-memory 的指引，从用户笔记中检索相关历史知识。

参数控制：
- 全面扫描模式：检索 10-15 条笔记，使用多种检索策略
- 聚焦模式：检索 5-8 条笔记，集中在特定主题

### 第三步：运行想法连接

将检索结果传递给 ie-connect-dots，按照其指引进行推理分析。

参数控制：
- 默认启用全部三种能力（聚类、连接、模式识别）
- 如果检索结果不足 3 条，跳过聚类，只做连接假设

### 第四步：运行洞见生成

将推理结果传递给 ie-generate-insight，按照其指引生成灵感内容。

参数控制：
- 默认生成 2-3 个洞见 + 2-3 个探索建议
- 如果有明显的时间演化线索，额外生成想法演化路径

### 第五步：注意力管理与呈现

在向用户呈现结果之前，进行注意力管理判断。

## 主动触发协议

当 Agent 主动判断用户可能从历史笔记中受益时，遵循以下协议：

### 第一步：快速预检

不要立即运行完整流水线。先用 ie-retrieve-memory 做一次轻量检索（2-3 条笔记），快速判断是否有相关内容。

### 第二步：轻量提示

如果预检发现了相关笔记，用一句话提醒用户，格式如下：

```
[灵感提示] 你之前在「[笔记标题]」等 [N] 条笔记中讨论过相关的想法，要我帮你深入看看有什么启发吗？
```

关键原则：
- 只用一句话，不展开分析
- 提及具体的笔记标题，让用户判断是否相关
- 以问句结尾，把决定权交给用户
- 不打断用户正在进行的工作流

### 第三步：等待用户响应

- 用户说「好的」「看看」等肯定回复 --> 运行完整流水线
- 用户忽略或说「先不用」 --> 不再追问，继续当前工作
- 用户说「稍后提醒我」 --> 记住这个话题，在合适的时机再次轻量提示

### 主动触发的克制原则

- 同一个对话中，主动提示最多出现 2 次，避免干扰用户
- 如果用户已经拒绝过一次主动提示，降低后续触发的敏感度
- 当用户明显处于专注执行状态（连续发出具体指令）时，不要插入提示

## 注意力管理

灵感引擎不应该用信息轰炸用户。呈现洞见之前，考虑以下因素：

### 呈现判断条件

只有同时满足以下条件时，才呈现完整的洞见结果：

1. **新颖性**：洞见中包含用户可能没有意识到的连接（不是显而易见的）
2. **置信度**：至少有一个连接假设的置信度为"中"或"高"
3. **可行动性**：至少有一个探索建议是可以立即行动的

### 不同情况的处理

- **三个条件都满足**：呈现完整结果（洞见 + 探索建议 + 演化路径）
- **只满足部分条件**：精简呈现，只展示满足条件的部分
- **没有有价值的发现**：坦率告知用户，并建议调整检索方向或扩大范围

### 呈现格式

向用户呈现时，使用以下结构：

```
灵感报告
===

检索范围：[使用的关键词/主题，涉及多少条笔记]

---

[洞见1]
[洞见2]（如果有）

---

下一步探索建议
[建议列表]

---

[想法演化路径（如果有）]

---

说明：以上洞见基于对你过去笔记的分析，是可能的启发方向，不代表确定性结论。
```

## 架构总览

```
ie-engine（本技能 - 编排与注意力管理）
  |
  +-- ie-retrieve-memory（记忆层 - 检索历史知识）
  |
  +-- ie-connect-dots（推理层 - 分析想法关系）
  |
  +-- ie-generate-insight（洞见层 - 生成灵感内容）
```

每个分层技能都可以独立使用。本技能的价值在于：
1. 提供一键式的完整流水线体验
2. 管理层间数据传递
3. 进行注意力管理，控制输出质量
