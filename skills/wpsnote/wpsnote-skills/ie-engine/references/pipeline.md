# 灵感引擎流水线详细定义

本文件定义了灵感引擎完整流水线的详细参数和运行模式。
被 ie-engine 的 SKILL.md 引用，在需要了解流水线细节时阅读。

---

## 流水线模式

### 模式一：全面扫描

适用于用户想要广泛回顾某个领域的全部相关笔记。

配置：
```
记忆检索
  关键词数量：3-5 个
  检索策略：关键词 + 标签 + 时间，多策略并行
  返回笔记数：10-15 条
  每条摘要长度：150-200 字

想法连接
  启用能力：聚类 + 连接假设 + 长期模式
  连接假设数量上限：5 个
  长期模式数量上限：3 个

洞见生成
  洞见数量：3 个
  探索建议数量：3 个
  演化路径：生成（如果有足够的时间跨度数据）
```

### 模式二：聚焦探索

适用于用户围绕一个具体想法寻找灵感。

配置：
```
记忆检索
  关键词数量：1-2 个（精确匹配）
  检索策略：关键词优先
  返回笔记数：5-8 条
  每条摘要长度：100-150 字

想法连接
  启用能力：连接假设（核心）+ 聚类（辅助）
  连接假设数量上限：3 个
  长期模式：跳过

洞见生成
  洞见数量：2 个
  探索建议数量：2 个
  演化路径：不生成
```

### 模式三：时间回溯

适用于用户想看某个时间段内的思考轨迹。

配置：
```
记忆检索
  检索策略：时间维度优先
  返回笔记数：10-20 条（按时间排序）
  每条摘要长度：100 字

想法连接
  启用能力：长期模式（核心）+ 聚类（辅助）
  连接假设：只在聚类间高相关时生成

洞见生成
  洞见数量：2 个
  探索建议数量：2 个
  演化路径：生成（核心输出）
```

## 模式选择逻辑

根据用户输入自动选择模式：

```
用户提到具体时间段（"去年""上个月""2024年"）
  --> 模式三：时间回溯

用户提到具体想法或主题（"关于知识图谱""那个截图解析的想法"）
  --> 模式二：聚焦探索

用户表达宽泛（"看看有什么启发""connect the dots""全面回顾"）
  --> 模式一：全面扫描

无法判断
  --> 询问用户偏好，默认使用模式一
```

## 层间数据传递规范

### 记忆层 --> 推理层

传递内容：
```
检索条件摘要
笔记列表（每条包含：标题、笔记ID、创建时间、标签、核心内容摘要）
检索置信度
```

### 推理层 --> 洞见层

传递内容：
```
语义聚类结果（聚类名称、每个聚类包含的笔记、聚类间关系）
连接假设列表（连接类型、相关想法、推理依据、置信度）
长期模式列表（主题名称、出现频率、演变趋势、代表笔记）
分析元信息（输入笔记数量、时间覆盖范围、整体置信度）
```

## 异常处理

### 检索结果过少

如果记忆层只返回 0-1 条笔记：
1. 尝试放宽检索条件（扩展关键词、移除标签限制）
2. 如果仍然不足，告知用户当前主题在历史笔记中没有足够的相关内容
3. 建议用户提供更多关键词或换一个主题

### 检索结果过多

如果记忆层返回超过 15 条笔记：
1. 按相关性排序，取前 15 条
2. 告知用户总共找到多少条，只分析了最相关的部分

### 推理结果无有价值的连接

如果推理层没有发现有意义的连接假设：
1. 跳过洞见生成
2. 向用户展示聚类结果（至少让用户看到笔记的分类）
3. 建议扩大检索范围或尝试不同的主题切入点
