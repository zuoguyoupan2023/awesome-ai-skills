# 洞见层：灵感生成与探索建议

本技能负责将推理层的分析结果转化为人类可读的洞见、探索建议和想法演化路径。
这是灵感引擎（Inspiration Engine）三层架构中的最上层，直接面向用户产出灵感内容。

## 适用场景

当需要将分析结果转化为可行动的灵感内容时使用，典型场景包括：

- 被 ie-engine 编排器调用，处理 ie-connect-dots 的推理结果
- 用户看到一组分析结果，想要更易读的洞见解读
- 用户想知道接下来可以探索什么方向
- 用户想回顾某个想法的演化历程

## 不适用场景

以下场景不应使用此技能（避免误触发）：

- 检索或搜索笔记（使用 ie-retrieve-memory）
- 分析笔记之间的关系（使用 ie-connect-dots）
- 撰写文章或创作内容（直接写作即可）
- 生成会议纪要或文档摘要（使用专门的写作工具）

---

## 三种生成能力

### 能力一：洞见生成

将推理层发现的连接和模式转化为有启发性的洞见文本。

好的洞见应该满足以下标准：
- **出乎意料**：揭示用户之前没有注意到的联系
- **有依据**：基于真实的笔记内容，而非空泛推测
- **可行动**：指向具体可以做的事情
- **简洁有力**：一段话说清一个洞见

生成方法：
1. 从推理层的连接假设中，选择置信度为"中"或"高"的条目
2. 将连接假设转化为自然语言的洞见描述
3. 引用原始笔记内容作为支撑
4. 点明这个洞见的潜在价值

输出格式：
```
洞见 1：[洞见标题]

[2-4句话的洞见正文，说明发现了什么、为什么重要]

依据：
  - 来自笔记「[笔记标题]」：[引用内容片段]
  - 来自笔记「[笔记标题]」：[引用内容片段]

潜在行动：[一句话描述可以基于此洞见做什么]
```

### 能力二：探索建议

基于分析结果提出下一步可以探索的方向。

探索建议的三个层次：
- **延伸型**：沿着现有主题继续深入
- **交叉型**：将两个不同领域的想法结合起来
- **反转型**：挑战现有假设，从反面思考

生成方法：
1. 从推理层的聚类结果和连接假设中提取可探索的方向
2. 每个方向给出具体的探索路径和第一步行动
3. 标注每个建议的类型和投入程度

输出格式：
```
探索建议
---

方向 1：[探索标题]
  类型：延伸型/交叉型/反转型
  为什么值得探索：[1-2句话]
  第一步行动：[具体可以做的一件事]
  预估投入：轻量（1小时内）/ 中等（数小时）/ 重度（需要数天）

方向 2：[探索标题]
  ...
```

### 能力三：想法演化路径

展示一个主题或想法在时间轴上的演变过程，帮助用户看清自己思考的发展脉络。

生成方法：
1. 从推理层的长期模式中选取有明显演化轨迹的主题
2. 按时间线排列相关笔记
3. 概括每个阶段的核心变化
4. 点明演化的整体趋势和可能的未来方向

输出格式：
```
想法演化：[主题名称]
===

阶段 1：[时间段] -- [阶段特征]
  代表内容：[笔记标题或内容要点]
  核心变化：[相比之前有什么新的认知]

阶段 2：[时间段] -- [阶段特征]
  代表内容：[笔记标题或内容要点]
  核心变化：[相比上一阶段的转变]

...

演化趋势：[整体趋势的一句话总结]
可能的下一步演化：[基于趋势的推测]
```

## 输入要求

本技能接受以下任一形式的输入：

1. **ie-connect-dots 的推理结果**（推荐，流水线标准路径）
   - 包含语义聚类、连接假设、长期模式
2. **用户直接提供的想法或分析结果**
   - 需要有足够的结构化信息来生成洞见

如果输入信息不足以生成有意义的洞见，坦率告知用户，建议先使用 ie-retrieve-memory 和 ie-connect-dots 获取更多素材。

---

## 输出质量原则

1. **诚实标注不确定性**：如果某个洞见的推理链条不够牢固，要明确说明是推测
2. **避免空泛**：每个洞见必须有具体的笔记引用支撑，不能只是泛泛而谈
3. **尊重用户判断**：洞见是"可能的启发"，不是"正确答案"，措辞上保持谦逊
4. **控制数量**：默认生成 2-3 个洞见 + 2-3 个探索建议，质量优先于数量
5. **不重复已知信息**：避免把用户已经明确知道的事情包装成"洞见"

## 与其他灵感引擎技能的关系

本技能是灵感引擎的 **洞见层**，在三层架构中处于最上层：

```
ie-generate-insight（洞见层）<-- 本技能，生成灵感内容
ie-connect-dots（推理层）   <-- 提供推理结果
ie-retrieve-memory（记忆层）<-- 提供原始素材
```

- **上游**：ie-connect-dots（推理层）提供聚类、连接和模式分析结果
- **下游**：无（本技能直接面向用户输出）
- **编排**：ie-engine 负责串联调用

本技能可以独立使用（用户直接给出分析结果让你生成洞见），也可以作为灵感引擎流水线的一部分。
