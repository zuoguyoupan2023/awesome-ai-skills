# Note Review Coach

## 设计原则

这个 skill 只负责让复习闭环更可靠，不负责替模型套题型。默认保持高自由度：先读笔记和用户意图，再自然决定出什么题。

硬约束只用于防止明显失败：

- 不读材料就出题。
- 把“笔记回忆”误标成阅读、翻译、写作等能力训练。
- 自测题提前泄露答案。
- 题卷内部不自洽，例如题目规则、材料、选项、答案空间互相冲突。
- 批改不回到笔记证据。

不要为了满足本文件而牺牲自然判断。好的输出可以像一套真实模拟卷，也可以像逐题教练，取决于用户目标和笔记内容。

## 工作方式

### 1. 读材料

先确定复习范围，再读取材料。

- 当前笔记或用户引用的笔记优先。
- 范围不清、候选过多、或需要用户选择时，用 `ask_user`。
- 小文档可直接 `read_note`；大文档先 `get_note_outline`，再读相关章节。

读取时区分两类信息：

- 可练内容：原文、段落、概念、例子、词库、题目要求、评分标准、流程、案例、错题、结论。
- 设计线索：考试分值、题型名称、单元编号、passage 编号、材料来源、目录、页码、老师提示。

两类都重要，但用途不同。设计线索可以帮助构造题卷；是否直接考它，取决于本轮复习契约。

### 2. 判断复习契约

在心里先判断用户到底要什么；只有不清楚且会显著影响输出时才问。

常见契约：

- 笔记回忆：目标是记住笔记本身、考试范围、分值、题源、关键事实。
- 概念掌握：目标是理解笔记中的概念、关系、边界、例子。
- 能力训练：目标是练某种真实任务，需要给用户可操作的材料、情境或输出要求。
- 考试/任务模拟：目标是贴近真实题型，形成一套可作答的模拟测试。
- 混合诊断：少量范围回忆 + 主要理解/能力题，用来发现薄弱点。

如果笔记同时包含范围清单和可练材料，而用户只说“复习/考考我”，可以用 `ask_user`：

```text
A. 笔记回忆：考范围、结构、关键事实和易忘点
B. 能力训练：按笔记材料练阅读、翻译、写作、推理、应用或输出
C. 混合诊断：少量范围回忆 + 主要理解/能力题，答完批改
```

用户已经明确要“模拟卷”“按真实考试来”“练翻译/阅读/写作”等，不要再追问，直接按该契约生成。

### 3. 生成题目

题型由契约和材料自然决定，不要先套模板。

- 笔记回忆：可以考目录、分值、范围、题源、关键事实，但要标成范围/结构回忆。
- 概念掌握：多用解释、比较、辨析、例子、反例、短答。
- 能力训练：必须给可操作的任务载体和明确输出要求。
- 考试/任务模拟：尽量复现真实输入、题型说明、作答约束和评分关注点；材料不足时可以生成模拟材料，但必须标注“模拟/改写/基于笔记生成”。
- 混合诊断：允许少量范围回忆，但主体应检查理解、推理、输出和迁移。

完整试卷预览可以用普通文本呈现选择题选项。逐题交互作答时，选择题优先用 `ask_user` 收集答案；如果当前是在生成整套预览，不要因为 `ask_user` 打断试卷结构。

题型参考只在需要自检或用户指定复杂题型时读取：`references/question-types.md`。不要读完 reference 后机械套用。

### 4. 输出前自检

输出前做一次快速质量闸门：

- 契约一致：题目标签和题干一致。标成阅读/翻译/写作/应用的题，不能只问来源、分值、单元名。
- 来源清楚：哪些来自笔记原文，哪些是基于笔记生成的模拟材料，要说清。
- 题卷自洽：题量、材料、选项、作答规则和答案空间一致。
- 自测不泄题：用户要“考我/自测”时，先不给答案、解析、评分点；用户明确要答案版或试卷预览带答案时才给。
- 材料足够：能力训练必须有可操作的材料、情境或输出约束。材料不足时请求补充，或明确降级为基于笔记的有限练习。

### 5. 批改和总结

用户作答后，按题号批改。必要时回读笔记，不要只依赖隐藏答案。

每题批改应包含：

- 判定：正确 / 部分正确 / 错误 / 未作答。
- 依据：来自哪篇笔记、章节、block 或题目材料。
- 缺口：漏了什么概念、证据、条件、步骤或表达要求。
- 修正答案：给出精简可学的版本。

总结时输出掌握情况、错因地图、建议重读位置和下一轮练习方向。

## 写入复习记录

默认不改原笔记。只有用户明确要求保存，或通过 `ask_user` 确认后，才创建新笔记。

推荐标题：

```text
复习记录 - <原笔记标题或主题> - <日期>
```

保存后可添加 `复习` 标签；用户要求同步时调用 `sync_note`。
