# 标签体系设计方法论

当用户的已有标签极少（3 个以下）、无法判断分类习惯时，根据笔记内容选择以下方法从头设计标签体系。

---

## 方法一：层级分类法

适用场景：知识积累型、长期归档型笔记库（个人百科、读书笔记、学习资料）

标签结构示例：

```
工作/产品
工作/技术
工作/会议
学习/读书
学习/英语
生活/健身
生活/理财
```

核心规则：
- 标签以**名词**为主，按主题从大到小嵌套
- 先确定 3-5 个顶层分类，再根据内容扩展子层级
- 同一个主题下的内容，尽量归入已有标签，不轻易新建顶层

---

## 方法二：P.A.R.A. 分类法

适用场景：行动导向型笔记库（项目制工作、任务管理）

标签结构（固定四个顶层）：

```
项目   ← 有明确目标和截止日期的任务（如 项目/产品改版）
领域   ← 需要持续维护的责任领域（如 领域/健康、领域/团队管理）
资源   ← 可能有用的参考资料（如 资源/竞品分析）
归档   ← 已完成或不再活跃的内容
```

核心规则：
- 只使用这四个顶层，不在其下建立多级嵌套（`项目/产品改版` ✅，`项目/工作/产品改版` ❌）
- 内容随状态变化在四个分类间迁移（项目完成 → 移入归档）

---

## 方法三：实体/场景法

适用场景：以人、事、地为核心的笔记库（销售/BD 的客户管理、项目跟进、高频出差人员）

标签结构示例：

```
按客户：公司A、公司B、张总
按地点：上海、北京出差
按项目：2024招标、Q1大促
```

核心规则：
- 标签是**具体的人名、公司名、地点、项目名**，高度具象
- 通常与其他方法结合使用（如：实体标签标注关系人 + 层级标签标注主题）
- 标签生命周期较短，项目结束或关系变化后及时清理

---

## 选型建议

- 根据笔记内容的整体特征和用户的使用场景推断最合适的方法
- 不确定时，默认推荐**层级分类法**（适应性最广，最易上手）
- 多种方法可以并列共存，但不要互相嵌套（如：层级分类管主题 + 实体标签标注关联人，各自独立）
- 在给出标签整理方案时，**明确说明选择了哪种分类方法及理由**，让用户确认
